# Access版メニュー導線からCloud版業務メニューへの対応

## 方針

Access版の見た目やボタン配置を完全再現するのではなく、フォーム遷移から読み取れる導線をCloud版に移す。

## Access版で確認できる主な導線

- メインメニューから会計管理、マスター、ユーティリティ、データ選択、データ保存系へ遷移する。
- 会計管理から現金出納帳、預金出納帳、経費帳、仕訳日記帳、仕訳伝票選択、月次残高推移系へ遷移する。
- 各画面の戻るボタンはメインメニューまたは会計管理へ戻る。
- データ変更画面からデータ年度繰越へ遷移する。
- ユーティリティからバックアップ復元、データ年度繰越へ遷移する。
- 決算書作成系は会計管理へ戻る導線を持つ。

## Cloud版の対応

- /books はAccess版のデータ選択に近づけ、帳簿の選択・状態確認・新規作成・年度締めへの入口だけに寄せる。
- 各業務への大量の入口は /work-menu に集約する。
- 軽量版の帳簿一覧を正式な resources/views/books/index.blade.php として扱う。
- 旧い大量ボタン版は resources/views/books/index_legacy.blade.php に退避し、必要になった場合だけ参照する。
- 見た目や並び順は後工程で調整し、まずは導線をAccess版に近づける。

- /work-menu をAccess版メインメニュー相当の入口にする。
- /accounting-menu をAccess版の FN_会計管理 相当の入口にする。
- /work-menu には会計画面を大量に並べず、会計管理メニューへの入口だけを置く。
- /accounting-menu には仕訳伝票選択、現金出納帳、預金出納帳、経費帳、仕訳日記帳、月次推移表など、FN_会計管理で確認できた導線を中心に置く。
- 決算・申告、年度繰越、賃貸管理、消費税は別メニュー扱いにする。
- /books は帳簿一覧・帳簿選択に寄せ、各業務への大量ボタンは /work-menu 側へ移す。
- /data-menu をAccess版のデータ選択・データ新規作成・データ変更・データ年度繰越相当の入口にする。
- /master-menu をAccess版のマスタ相当の入口にする。
- /utility-menu をAccess版のユーティリティ相当の入口にする。
- /work-menu にはデータ、会計管理、マスタ、ユーティリティの詳細ボタンを大量に置かず、それぞれの下位メニューへの入口を置く。
- バックアップ・復元は現時点では未実装表示とし、後続で実装する。
- 年度繰越はAccess版でデータ変更・ユーティリティ双方から入れる導線があるため、/data-menu と /utility-menu の両方に入口を置く。
- /rental-menu を賃貸管理、物件台帳、契約者、賃貸条件、退去処理の入口にする。
- /payment-menu を入金予定、入金実績、賃貸入金仕訳、入金差額処理、預り金処理の入口にする。
- /tax-menu を決算整理、不動産所得決算書、青色・白色プレビュー、消費税、物件別損益確認の入口にする。
- /output-menu をAccess版の帳票・印刷指定に相当する入口にする。
- /work-menu は各業務の詳細ボタンを置かず、下位メニューへの入口を並べる構成に近づける。
- これにより、Access版の「メインメニュー → 各業務メニュー → 個別画面」に近い階層になる。
- 会計管理、賃貸管理、入金管理、決算・申告、年度繰越、マスタ、帳票・出力に分類する。
